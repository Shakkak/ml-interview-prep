# Hidden Markov Models

---

## Fundamental

Many real sequences have an underlying state that drives what we observe — a person's activity drives their phone sensor readings; a word's part of speech drives its likely neighbors; a market regime drives asset returns. A **Hidden Markov Model (HMM)** captures this: the hidden state follows a Markov chain (each state depends only on the previous one), and observations are noisy signals from the current hidden state. HMMs were the backbone of speech recognition for decades and remain a core model for biological sequences and time-series segmentation.

### Model Definition

A **Hidden Markov Model (HMM)** models a sequence of observations $x_{1:T}$ generated by an unobserved (hidden) Markov chain of states $z_{1:T}$.

**Components:**
- $K$ hidden states, $M$ possible observations
- **Initial distribution:** $\pi_k = p(z_1 = k)$
- **Transition matrix:** $A_{jk} = p(z_t = k \mid z_{t-1} = j)$ — rows sum to 1
- **Emission distribution:** $B_k(x) = p(x_t = x \mid z_t = k)$ — discrete (categorical) or continuous (Gaussian)

The joint probability factorizes:

$$p(z_{1:T}, x_{1:T}) = \pi_{z_1} \prod_{t=2}^T A_{z_{t-1}, z_t} \prod_{t=1}^T B_{z_t}(x_t)$$

**Why "hidden":** we observe $x_{1:T}$ but not $z_{1:T}$. The task is to infer $z$ from $x$ and/or learn $\theta = (\pi, A, B)$.

## Intermediate

### Three Fundamental Problems

**1. Evaluation** — $p(x_{1:T} \mid \theta)$: likelihood of an observation sequence. Used for model comparison or anomaly detection.

**2. Decoding** — $\arg\max_{z_{1:T}} p(z_{1:T} \mid x_{1:T})$: most likely hidden state sequence. Used for sequence labeling.

**3. Learning** — $\arg\max_\theta p(x_{1:T} \mid \theta)$: find parameters that best explain observations.

### Forward-Backward Algorithm

Define the **forward variable** $\alpha_t(k) = p(x_{1:t}, z_t = k)$ and **backward variable** $\beta_t(k) = p(x_{t+1:T} \mid z_t = k)$.

**Forward recursion:**
$$\alpha_t(k) = B_k(x_t) \sum_j \alpha_{t-1}(j) A_{jk}, \quad \alpha_1(k) = \pi_k B_k(x_1)$$

**Backward recursion:**
$$\beta_t(j) = \sum_k A_{jk} B_k(x_{t+1}) \beta_{t+1}(k), \quad \beta_T(k) = 1$$

**Posterior:**
$$p(z_t = k \mid x_{1:T}) = \frac{\alpha_t(k)\beta_t(k)}{\sum_j \alpha_t(j)\beta_t(j)}$$
$$p(z_t = j, z_{t+1} = k \mid x_{1:T}) \propto \alpha_t(j) A_{jk} B_k(x_{t+1}) \beta_{t+1}(k)$$

Both run in $O(TK^2)$ time.

### Viterbi Algorithm

Finds the most probable state sequence via dynamic programming. Define $\delta_t(k) = \max_{z_{1:t-1}} p(z_{1:t-1}, z_t = k, x_{1:t})$:

$$\delta_t(k) = B_k(x_t) \max_j \delta_{t-1}(j) A_{jk}$$

Track the argmax (backpointer) at each step; recover the sequence by backtracking from $t=T$. Same $O(TK^2)$ complexity as forward-backward but with max instead of sum.

**Viterbi vs MAP marginals:** Viterbi gives the jointly most probable path. Forward-backward gives the most probable state at each $t$ marginally — these can disagree (the individually best states may form an impossible sequence).

### Baum-Welch: EM for HMMs

**Baum-Welch** is the EM algorithm applied to HMMs.

**E-step:** run forward-backward to compute:
- $\gamma_t(k) = p(z_t = k \mid x_{1:T}, \theta^\text{old})$ — state occupancy
- $\xi_t(j,k) = p(z_t=j, z_{t+1}=k \mid x_{1:T}, \theta^\text{old})$ — transition occupancy

**M-step:** update parameters with their MLEs under the expected sufficient statistics:
$$\hat{\pi}_k = \gamma_1(k)$$
$$\hat{A}_{jk} = \frac{\sum_t \xi_t(j,k)}{\sum_t \gamma_t(j)}$$
$$\hat{B}_k(x) = \frac{\sum_t \gamma_t(k) \cdot \mathbf{1}[x_t=x]}{\sum_t \gamma_t(k)}$$

For Gaussian emissions: $\hat{\mu}_k = \frac{\sum_t \gamma_t(k) x_t}{\sum_t \gamma_t(k)}$, $\hat{\Sigma}_k$ similarly.

## Advanced

### Connections and Extensions

**HMM vs SSM:** linear Gaussian SSMs (Kalman filter) are the continuous-state continuous-emission analogue. The Kalman filter implements the forward pass; the RTS smoother implements forward-backward for Gaussian states.

**HMM vs RNN:** RNNs (see [[rnn-lstm]]) are discriminative sequence models with a continuous hidden state updated by a nonlinear function. HMMs are generative and have a discrete hidden state updated by a linear table. RNNs are more expressive but require labeled data; HMMs can be trained unsupervised.

**HMM applications:**
- Speech recognition (acoustic model over phoneme states)
- POS tagging and NER (state = grammatical tag)
- Genomics (CpG island detection, gene finding)
- Financial regime detection

**Higher-order HMMs:** $p(z_t \mid z_{t-1}, z_{t-2}, \ldots)$ — can be converted to first-order by expanding the state space.

**Factorial HMMs:** multiple independent hidden chains generate observations jointly — exponential state space, but inference is tractable via mean-field approximation.
